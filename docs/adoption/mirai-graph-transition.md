# Mirai Graph Naming Transition

Status: historical naming transition note

Mirai Graph has completed its public naming transition. This document records
the current naming model and the boundaries for adopters.

The current naming model is:

- `Mirai` is the broader AI direction.
- `Mirai Graph` is the public standard, model and methodology.
- `mirai-graph` is the repository, package and primary CLI slug.
- `EOG` remains a scientific continuity term for earlier research framing.

The strategic goal of the rename is not cosmetic. It separates the future-facing
Mirai direction from earlier working labels and positions Mirai Graph as the
public graph standard for governed AI work: AI employees, projects,
organizations and research systems coordinated through state, evidence, gates,
feedback and controlled evolution.

## What Changed

- Package name: `mirai-graph`.
- Primary CLI: `mirai-graph`.
- Preferred package manifest: `mirai-graph-package.json`.
- Public docs and examples use Mirai Graph naming.

## Current Command Surfaces

The supported current command surfaces are:

- `npx mirai-graph ...` for adopter projects;
- `node packages/cli/mirai-graph.js ...` for repository checkout maintenance;
- `node packages/cli/mirai_graph.js ...` as an underscore alias;
- `node packages/cli/validate-mirai-graph.js ...` for direct validator calls;
- `mirai-graph-package.json`

The underscore CLI is an alias for local environments that prefer identifier
style command names. It is not a separate standard.

## Migration Steps

1. Use `mirai-graph-package.json` for package manifests.
2. Use `npx mirai-graph validate .` in adopter projects.
3. Use direct validator calls from repository checkout only when maintaining
   Mirai Graph itself.
4. Keep generated reports as historical evidence; do not rewrite evidence
   payloads unless they are regenerated by the current tooling.

## Schema Namespace

Existing schema `$id` values may still use the previous namespace during the
transition. Mirai Graph will move schema namespaces only after the public domain
decision is confirmed. Schema `$id` changes should be treated as a compatibility
event, not as a cosmetic text edit.

## Boundary

The rename does not change the standard semantics:

- generated context is not canonical state;
- evidence and proposals do not authorize canonical updates;
- feedback does not automatically change behavior;
- readiness is not execution;
- command aliases are not new standards.
