# Mirai Graph v0.1.0-alpha.9

Status: published GitHub prerelease; npm publication pending registry
authentication.

## Theme

Public naming consolidation under Mirai Graph.

This alpha consolidates the public standard, package, repository and CLI
identity around the `mirai-graph` slug.

## Included Since v0.1.0-alpha.8

### Rename Transition

- Public standard name is now Mirai Graph.
- Package name is now `mirai-graph`.
- Primary CLI entrypoint is now `packages/cli/mirai-graph.js`.
- Preferred package manifest is now `mirai-graph-package.json`.
- The underscore CLI alias `packages/cli/mirai_graph.js` is available for local
  environments that prefer identifier-style command names.

### Command Surface

- `packages/cli/mirai-graph.js` is the primary local CLI.
- `packages/cli/mirai_graph.js` delegates to the primary CLI as an underscore
  alias.
- Validators use `mirai-graph-package.json`.
- `npm test` includes a smoke check for the primary CLI, underscore alias and
  current manifest filename.

### Documentation

- Added a transition guide:
  `docs/adoption/mirai-graph-transition.md`.
- Updated README, CLI docs, citation metadata, changelog and release navigation
  for Mirai Graph naming.

### Release State

- GitHub repository: `zabarov/mirai-graph`.
- GitHub release:
  `https://github.com/zabarov/mirai-graph/releases/tag/v0.1.0-alpha.9`.
- Git tag: `v0.1.0-alpha.9`.
- NPM package: pending publication until npm registry authentication is
  available.

## Claim Boundary

This release should claim only:

```text
Mirai Graph v0.1.0-alpha.9 consolidates public naming, package identity and
local CLI surfaces around Mirai Graph.
```

It should not claim:

- schema namespace ownership has been finalized;
- npm package publication has happened;
- historical evidence payloads were regenerated;
- peer-reviewed proof of effectiveness.

## Known Limits

- NPM publish remains pending registry authentication.
- Schema `$id` namespace migration remains pending domain confirmation.
- The underscore CLI alias is an implementation convenience, not a separate
  public standard.
