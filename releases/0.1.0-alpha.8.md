# GrowGraph v0.1.0-alpha.8

Status: release preparation notes

## Theme

Adopter kit and executable technology-quality control.

This alpha keeps GrowGraph CLI-first and public-safe while making adoption
more practical through starter templates, organization-governance pilot
coverage, a unified CLI wrapper and a technology quality feedback gate that
checks whether work followed the declared process before acceptance or release.

## Included Since v0.1.0-alpha.7

### Adopter Kit

- Added starter templates for software projects, research programs,
  AI employees and organization governance.
- Added unified `growgraph` CLI wrapper for common validation, report and
  process-transition explanation commands.
- Added organization-governance pilot coverage.
- Added Evidence Package v0.2 as the current public research evidence
  entrypoint.

### Technology Quality Feedback

- Added `standard/technology-quality-feedback.md`.
- Added `schemas/technology-quality-feedback.schema.json`.
- Added synthetic example:
  `examples/technology-quality-feedback-minimal/`.
- Added CLI mode:
  `node packages/cli/validate-growgraph.js technology-quality-feedback <file>`.
- Added `validate:technology-quality-feedback` and
  `test:technology-quality-feedback-negative`.
- Added fail-closed negative fixtures for blocking findings, missing process
  steps, unclassified findings, missing Kaizen routes, unresolved spec gaps,
  missing feedback reports and evidence-as-authorization errors.

### Process-Control Integration

- Process-control docs now state that significant acceptance, approval or
  release transitions should require technology quality feedback.
- The implementation-control profile now includes technology contracts,
  technology quality feedback reports and quality findings.
- The process-control contract schema supports an optional
  `technology_quality_feedback_policy`.

## Validation

Validate with:

```bash

```

Targeted alpha.8 checks:

```bash
npm run validate:technology-quality-feedback
npm run test:technology-quality-feedback-negative
npm run validate:process-control
npm run validate:cli-unified
```

## Claim Boundary

This release should claim only:

```text
GrowGraph v0.1.0-alpha.8 adds public alpha readiness for adopter templates,
unified CLI entrypoints, organization-governance pilot coverage and executable
technology-quality feedback gates.
```

It should not claim:

- peer-reviewed proof of effectiveness;
- production runtime safety;
- autonomous execution readiness;
- graph-only runtime readiness;
- automatic acceptance from passing tests;
- automatic canonical update from generated context, feedback, evidence,
  runtime results or proposals.

## Known Limits

- CLI remains reference tooling, not a full SDK.
- Technology quality feedback validates report shape and boundary rules; it
  does not prove semantic correctness by itself.
- Public examples remain synthetic and require external replication.
