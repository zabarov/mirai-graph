# Mirai Graph v1.0.0-rc.2

Status: release candidate prepared in repository; tag, GitHub Release and npm
publish require separate release execution.

## Theme

Self-Service Onboarding And Public Documentation Hardening.

This release candidate builds on `v1.0.0-rc.1` by making Mirai Graph easier to
adopt from an external project checkout while keeping the same safety boundary:
Mirai Graph validates graph-managed structure, evidence, process state and
governance gates; it does not execute production actions or authorize canonical
updates by itself.

## Added

- Self-service onboarding CLI flow:
  - `mirai-graph init`;
  - `mirai-graph detect`;
  - `mirai-graph bootstrap`.
- Synthetic self-service onboarding fixtures for:
  - project without a graph;
  - existing graph;
  - stale or incomplete graph;
  - generated bootstrap proposal.
- GitHub Action starter for validating a Mirai Graph package.
- "Connect A Project In 15 Minutes" tutorial.
- Developer-first entry guide explaining concrete problems, created files,
  expected outputs and AI-assisted development use cases.
- Documentation map in `docs/README.md`.
- Public-safe Societal Governance proposal profile and synthetic fixtures.
- Negative fixtures for proposal-level societal-governance safety boundaries.

## Changed

- Public README is now a shorter landing page with npm-first quickstart,
  developer-first problem framing, created-file expectations, role paths and a
  "what should I read first" table.
- Adoption docs now distinguish npm usage from contributor checkout usage.
- Release notes and roadmap now reflect self-service onboarding as part of the
  release-candidate surface.
- Societal Governance is explicitly marked proposal/experimental, not a core
  `1.0.0` conformance profile.


## Claim Boundary

This release candidate supports the claim that Mirai Graph has a coherent,
validator-backed public standard shape and a usable self-service onboarding
path for external review.

It does not claim:

- peer-reviewed scientific proof;
- broad external validity;
- production autonomous execution safety;
- final stable `1.0.0`;
- core conformance for the proposal-level Societal Governance profile;
- automatic canonical updates from generated context, evidence, feedback,
  cockpit metrics or proposals;
- npm package publication until npm release-state checks confirm publication.

## Release Execution Boundary

This file is release preparation only. Creating a git tag, GitHub Release or npm
publication must happen as a separate explicit release execution step.
