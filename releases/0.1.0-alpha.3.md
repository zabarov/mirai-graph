# Mirai Graph 0.1.0-alpha.3

Release type: profile conformance, semantic review and release-process alpha

## Summary

This release strengthens Mirai Graph as a public research-engineering package.
It adds a second independent implementation pilot, executable profile
conformance checks, semantic-completeness review artifacts and a public release
process.

## Added

- Independent implementation pilot 002 for a synthetic software specification
  workflow.
- Software-specification profile documentation.
- Profile conformance fixture guide.
- Validator checks for:
  - relation id consistency with source, type and target;
  - profile object-kind conformance;
  - profile relation-type conformance.
- Negative fixtures for profile and relation-id failures.
- Semantic-completeness review artifacts for:
  - synthetic benchmark context pack;
  - software-specification pilot context pack.
- Safety-context inclusion policy for context-pack generation.
- Public release process and GitHub Release notes policy.

## Validation

Run:

```bash
npm run release:check
```

Expected:

- package, seed, profile and pilot validation passes;
- context-pack generation succeeds;
- readiness score remains reproducible;
- synthetic benchmark remains reproducible;
- negative fixtures fail as expected;
- whitespace check passes.

## Evidence Notes

- Pilot 002 supports the limited claim that a valid Level 1 Mirai Graph package
  can be created for a small synthetic software specification.
- Semantic review of pilot 002 improved from `13/16` to `15/16` after
  safety-sensitive context inclusion.
- The synthetic benchmark semantic review records `15/16`.

## Important Limitations

- Evidence remains synthetic and public-safe.
- This release does not prove real-world productivity improvement.
- Context-pack selection still lacks per-object explanations and relevance
  scores.
- Semantic review artifacts are maintained review records, not automated
  consensus or peer review.
