# Mirai Graph 1.0.0-rc.6

Status: release-candidate
Date: 2026-06-27

## Summary

`1.0.0-rc.6` consolidates the anti-drift / quality-control contour across
existing Mirai Graph release-candidate layers:

```text
Semantic Intent Resolution
-> Process and Technology Control
-> Dynamic Episode Trace
-> Goal Vector Quality Control
-> Technology Quality Feedback
-> Kaizen / Replay / Regression Candidate
```

This release does not introduce a new authorization layer. It presents already
validator-backed public artifacts as one coherent model for inspecting whether
AI-assisted work started from the right intent, followed the declared process,
recorded what happened, advanced the final outcome and routed drift findings to
correction.

## Consolidated Standard Surface

- [Semantic Intent Resolution](../standard/semantic-intent-resolution.md)
  checks the input decision before execution starts.
- [Dynamic Episode Layer](../standard/dynamic-episode-layer.md) records what
  happened in one governed event and why a path was selected or blocked.
- [Goal Vector Quality Control](../standard/goal-vector-quality-control.md)
  checks whether meaningful work moved toward the final outcome.
- [Technology Quality Feedback](../standard/technology-quality-feedback.md)
  checks whether the declared technology or method was followed before
  acceptance or release.
- [Process Control Contract](../standard/process-control-contract.md) keeps
  launch, transition, evidence and false-transition guards explicit.

## Publication Foundations

This release adds public-safe foundations for two article candidates:

- [Semantic Intent And Goal Vector Control](../publications/semantic-intent-goal-vector-control/README.md)
- [Observable Self-Correcting AI Workflows](../publications/observable-self-correcting-ai-workflows/README.md)

Both foundations are scaffolds, not submitted papers. They define research
questions, hypotheses, artifact inventory, metrics, limitations and next
evidence needed before publication claims.

## Claim Boundary

This release validates public shapes, examples, negative fixtures and
documentation alignment for the consolidated anti-drift contour.

It does not prove broad real-world drift reduction, does not prove autonomous
production safety, does not authorize live or destructive actions, does not
update canonical graph state from generated briefs, traces, evidence, audits or
proposals and does not claim peer-reviewed scientific proof.

Sanitized pilot/replay evidence is still required before stronger effectiveness
claims.
