# Mirai Graph v0.1.0-alpha.7

Status: release preparation notes

## Theme

Readiness jump for explainable executable process control.

This alpha makes Mirai Graph easier to inspect without maintainer guidance by
adding process-transition decision explanations, a deterministic local
playground report, baseline-comparison evidence and an additional public-safe
pilot outside the original software and AI-employee examples.

## Included Since v0.1.0-alpha.6

### Process-Transition Explanations

- Process-transition validator JSON output now includes an `explanation`
  section.
- Explanation output records current and target states, matched transition,
  required and provided evidence, missing evidence, launch-record status,
  terminal/Kaizen decision and false-transition guard summary.
- Markdown process-transition output is now a transition decision report.
- Passing and failing explanation fixtures are committed under
  `examples/process-transition-minimal/results/`.
- `validate:process-transition-report` checks explanation-critical fields.

### Playground Report

- Playground remains CLI-first and deterministic.
- `playground:report` generates the same report as `playground:demo`.
- `playground/demo-report.md` now shows:
  - seed;
  - graph package;
  - context pack;
  - launch record;
  - transition explanation;
  - evidence;
  - Kaizen;
  - baseline comparison.
- `validate:playground-report` keeps the stored demo report aligned with the
  generated output.

### Baseline Comparison Evidence

- Added `schemas/baseline-comparison-result.schema.json`.
- Added public-safe synthetic result artifact at
  `benchmarks/synthetic-context-reduction-v0/results/baseline-comparison-result.json`.
- Added `validate:baseline-comparison`.
- Evidence docs now separate synthetic comparison, pilot evidence and
  scientific proof boundaries.

### Public-Safe Pilot Coverage

- Added independent implementation pilot 004:
  `pilots/independent-implementation-004-research-program/`.
- The pilot models a synthetic research-program governance workflow using the
  existing `project_management` profile.
- The pilot includes seed, package, graph objects, graph relations, gates,
  report and profile-conformance result.
- Public safety boundary excludes private research repositories, reviewer
  notes, handoff text, internal traces, secrets and customer or employee data.

### Release Packaging

- Updated package version to `0.1.0-alpha.7`.
- Updated README navigation, roadmap, playground docs, evidence docs,
  conformance fixture docs and changelog.

## Claim Boundary

This release should claim only:

```text
Mirai Graph v0.1.0-alpha.7 adds public alpha readiness for explainable
process-transition reports, deterministic local demos, bounded baseline
comparison evidence and wider public-safe pilot coverage.
```

It should not claim:

- peer-reviewed proof of effectiveness;
- production runtime safety;
- autonomous execution readiness;
- graph-only runtime readiness;
- universal superiority over document or task-tracker workflows;
- automatic canonical update from generated context, feedback, evidence,
  runtime results or proposals.

## Known Limits

- CLI remains reference tooling, not a full SDK.
- Playground is deterministic and local, not a hosted interactive environment.
- Baseline comparison is synthetic and requires replication.
- Public pilots show applicability, not broad external validity.
