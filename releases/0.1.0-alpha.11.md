# Mirai Graph v0.1.0-alpha.11

Status: prepared as a next alpha layer; tag, GitHub Release and npm publish are
out of scope for this batch.

## Theme

Instrumentation Layer.

Alpha.11 prepares Mirai Graph to model not only governed process transitions,
but the instruments used to steer them: development cockpit signals, feature
implementation traceability and multi-source quality feedback.

## Included Scope

### Development Cockpit

- New standard: `standard/development-cockpit.md`.
- New schema: `schemas/development-cockpit.schema.json`.
- New result validator mode: `development-cockpit`.
- Synthetic report in `examples/instrumentation-layer-minimal`.

The cockpit is an instrument panel. It can recommend a bounded next action, but
it cannot authorize execution, acceptance, release or canonical graph update.

### Feature Implementation Traceability

- New standard: `standard/feature-implementation-traceability.md`.
- New schema: `schemas/feature-implementation-traceability.schema.json`.
- New result validator mode: `feature-implementation-traceability`.

The traceability layer connects feature intent to implementation references,
tests, review, documentation and evidence. It keeps `specified`, `mapped`,
`implemented`, `tested`, `reviewed`, `accepted` and `released` separate.

### Multi-Source Quality Feedback

- New standard: `standard/multi-source-quality-feedback.md`.
- New schema: `schemas/multi-source-quality-feedback.schema.json`.
- New result validator mode: `multi-source-quality-feedback`.

The feedback layer combines tests, validators, cockpit signals, traceability,
human review and runtime observations into classified findings and transition
routes. Blocking findings stop the transitions they block.

### Synthetic Example

- New package: `examples/instrumentation-layer-minimal`.
- New npm checks:
  - `validate:development-cockpit`;
  - `validate:feature-implementation-traceability`;
  - `validate:multi-source-quality-feedback`;
  - `validate:instrumentation-layer`.

## Out Of Scope

- Runtime autonomy.
- Production execution.
- Automatic canonical updates from evidence, feedback, cockpit metrics or
  proposals.
- Private pilot payloads or private runtime traces.
- Peer-reviewed scientific proof.

## Validation

```bash
npm run validate:instrumentation-layer
npm test
npm run release:check
```

## Claim Boundary

This layer supports public alpha claims for observable process steering and
reproducible instrumentation artifacts.

It does not claim that synthetic cockpit scores predict real production
outcomes, that tests are acceptance, or that evidence/proposals authorize
canonical updates.
