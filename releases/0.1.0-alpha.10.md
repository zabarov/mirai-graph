# Mirai Graph v0.1.0-alpha.10

Status: planned release skeleton

## Theme

Adopter Workflow.

Alpha.10 should make Mirai Graph easier to apply without author-side guidance.
The release should turn the current standards, profiles, templates, examples
and CLI validators into a clearer self-service adoption path.

## Planned Scope

### Adopter Journey

- A concise profile-selection guide for developers, researchers, product teams
  and organization-governance adopters.
- A runnable path from starter template to validated package.
- A single practical workflow:

```text
choose profile -> copy starter -> fill graph package -> validate -> generate report -> launch work -> record evidence -> run feedback/kaizen
```

### CLI Usability

- Cleaner public command surface for common adopter tasks.
- Human-readable validation reports beyond process-transition checks.
- Starter workflow commands or documented command bundles.
- Import/export helper skeletons if they can stay implementation-independent.

### Adoption Evidence

- Clearer organization-governance validation report.
- Updated public-safe adopter examples.
- Research evidence notes that distinguish synthetic evidence, pilot evidence
  and scientific proof boundaries.

## Out Of Scope

- Runtime autonomy.
- Production execution.
- Automatic canonical updates from evidence, feedback or proposals.
- Private SIMAI/Larena/skill-federation payloads.
- Broad scientific proof claims.

## Done When

- A new adopter can pick a profile and run a complete local validation path.
- The CLI has a documented happy path for the starter templates.
- Validation reports are readable enough to support review without inspecting
  raw JSON first.
- Organization-governance adoption has a stronger public-safe example/report.
- `npm test` and `npm run release:check` pass.
