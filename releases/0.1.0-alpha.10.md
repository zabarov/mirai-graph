# Mirai Graph v0.1.0-alpha.10

Status: GitHub prerelease prepared; npm publication pending registry
authentication.

## Theme

Adopter Workflow.

Alpha.10 makes Mirai Graph easier to apply without author-side guidance. The
release turns the current standards, profiles, templates, examples and CLI
validators into a clearer self-service adoption path.

## Included Scope

### Adopter Journey

- A concise profile-selection guide for developers, researchers, product teams
  and organization-governance adopters.
- A runnable path from starter template to validated package.
- A single practical workflow:

```text
choose profile -> copy starter -> fill graph package -> validate -> generate report -> launch work -> record evidence -> run feedback/kaizen
```

### CLI Usability

- `mirai-graph choose-profile` for role/profile/starter selection.
- `mirai-graph adopter plan <role-or-profile>` for next-command guidance.
- `mirai-graph adopter report <template-dir>` for a readable starter report.
- `mirai-graph release state` for non-secret release-state checks across the
  package version, git tag, GitHub Release, npm registry and npm auth boundary.
- `validate:adopter-workflow` for command-level regression checks.
- `validate:release-state` for pre-tag release-state report regression checks.
- `mirai-graph release state --require-github-release` for post-GitHub-Release
  verification.
- Import/export helper skeletons remain a follow-up unless they directly
  support the adopter happy path.

### Adoption Evidence

- Clearer organization-governance validation report.
- Updated public-safe adopter examples.
- Research evidence notes that distinguish synthetic evidence, pilot evidence
  and scientific proof boundaries.

## Out Of Scope

- Runtime autonomy.
- Production execution.
- Automatic canonical updates from evidence, feedback or proposals.
- Private SIMAI/Larena/skill-federation payloads.
- Broad scientific proof claims.

## Done

- A new adopter can pick a profile and run a complete local validation path.
- The CLI has a documented happy path for the starter templates.
- Validation reports are readable enough to support review without inspecting
  raw JSON first.
- Organization-governance adoption has a stronger public-safe example/report.
- Release-state reporting separates GitHub Release readiness, npm registry
  state and npm authentication state.
- `npm test` and `npm run release:check` pass.

node packages/cli/mirai-graph.js release state --require-github-release --markdown

## Claim Boundary

This release supports public alpha claims for starter-template adoption,
role/profile selection, readable local reports and release-state visibility.

It does not claim npm publication, peer-reviewed proof, runtime autonomy,
production safety or automatic canonical updates from generated context,
evidence, feedback or proposals.
